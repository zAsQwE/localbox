## localbox (Offline Jackbox)

Возможность локально запускать серию игр Jackbox

Собственный игровой сервер (Ecast + Blobcast) на Node.js. Веб-интерфейс основан на **jackbox.tv** (английский) и **jackbox.ru** (русский).

tg = https://t.me/lacalbox (alt = https://telegram.me/lacalbox)

## Требования по системам

Сервер написан на **Node.js**, лаунчер — на **Python** (запускается через `python3 localbox_launcher.py`). Оба нужны на любой ОС.

### Windows

| Что                     | Обязательно    | Как поставить                                                                                                                                                                                                                                                                                                                                                          |
| -------------------------- | ------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Node.js**          | да                      | Скачать LTS-установщик с https://nodejs.org (`.msi`), установить (npm входит в комплект). Проверка в `cmd`: `node -v`.                                                                                                                                                                                                   |
| **Python 3**         | да                      | https://python.org → при установке отметить**«Add python.exe to PATH»**. tkinter входит в установщик Windows.                                                                                                                                                                                                                              |
| **mkcert**           | по желанию       | Для доверенного сертификата.`choco install mkcert` (если есть Chocolatey) или скачать `mkcert.exe` из релизов на GitHub, положить в PATH, затем `mkcert -install`. Без него будет самоподписанный серт (примешь предупреждение в браузере). |
| Права на порты | **не нужно** | Windows разрешает порты 80/443 без спец-действий. Кнопку «Разрешить порты» на Windows жать не надо.                                                                                                                                                                                                                 |

### Linux

Node.js по дистрибутивам:

```sh
# Debian / Ubuntu (свежий node):
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash - && sudo apt-get install -y nodejs
# Arch / CachyOS:
sudo pacman -S nodejs npm
# Fedora:
sudo dnf install nodejs npm
```

| --------------------------------- | ------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Node.js**                 | да                            | см. выше                                                                                                                                                                              |

## Быстрая установка

### Linux / macOS

После клонирования — одна команда ставит **всё** (Node.js, Python, ffmpeg, mkcert через пакетный менеджер, зависимости движка, клиент) — только недостающее:

```sh
./setup.sh          # всё + английский клиент
./setup.sh --ru     # + русский клиент (client-ru) с jackbox.ru
./setup.sh --run    # установить и сразу запустить
```

Запуск после установки — **`./start-server.sh`** (меню: обычный режим или с Додо Ре Ми, затем лаунчер).
Или напрямую: `cd launcher && python3 localbox_launcher.py`.

### Windows

Двойной клик по **`install.bat`** (поставит недостающее через `winget`: Node.js, Python, ffmpeg, git; установит зависимости; скачает клиент). Затем **`start-server.bat`** — откроет лаунчер.

> Если `install.bat` только что поставил Node/Python — закрой окно и запусти `install.bat` ещё раз (нужен обновлённый PATH). Без `winget` — поставь Node.js (nodejs.org) и Python (python.org, «Add to PATH») вручную и запусти снова.

Ниже — то же вручную и подробно.

## Первая настройка (подробно)

В репозитории **нет тяжёлых частей** (они в `.gitignore`): зависимостей движка (`node_modules`) и веб-клиентов (`client/`, `client-ru/`). Их нужно получить один раз.

**1. Получить проект**

```sh
git clone https://github.com/zAsQwE/localbox localbox
# или скачать ZIP и распаковать
```

**2. Проверить структуру папок** — рядом должны лежать:

```
localbox/
  server/        ← игровой сервер (наш)
  launcher/      ← лаунчер
  client-ru/  или  client/   ← веб-клиент для телефонов (см. шаг 4)
```

Если есть только `launcher/` — скопирован не весь проект.

**3. Установить Node.js** (см. «Требования» для своей ОС).

**4. Получить веб-клиент** (хотя бы одну папку). Сами клиенты в этот репозиторий **не входят** (это ассеты Jackbox) — их скачивают отдельно из исходных источников:

- **Английский клиент `client/`** — из стороннего репозитория [DdejjCAT/jackbox.tv](https://github.com/DdejjCAT/jackbox.tv) (полный набор ассетов):

  ```sh
  git clone https://github.com/DdejjCAT/jackbox.tv client
  ```
- **Русский клиент `client-ru/`** — качается с jackbox.ru встроенным скриптом:

  ```sh
  node tools/fetch-fun.js --host jackbox.ru --dst client-ru
  ```

  Можно ограничиться тонкой оболочкой: **недостающие текстуры движок докачает с jackbox.ru при первой загрузке и закэширует** (интернет нужен один раз, дальше офлайн).

**5. Зависимости движка** — установятся сами при первом запуске лаунчера, либо вручную:

```sh
cd server && npm i
```

**6. Запускать** — см. ниже (Windows GUI или Linux-сервер).

## Запуск — Windows (с интерфейсом)

1. Двойной клик по **`start-server.bat`** (или в папке `launcher`: `python localbox_launcher.py`).
2. В окне указать адрес: `127.0.0.1` (игра на этом же ПК) или локальный IP (для телефонов в сети).
3. Нажать **1. Сертификат**. (Кнопку «2. Разрешить порты» на Windows жать **не нужно**.)
4. Нажать **▶ Запустить**. В логе должны подняться порты 80/443/38202/38203.
5. Открыть в браузере **`https://адрес`**.

---

## Запуск — Linux (сервер, без GUI)

> Может быть небезопасно — для игры с друзьями, не как публичный сервис.

```sh
cd launcher
python3 localbox_launcher.py -ip=АЙПИ_СЕРВЕРА            # с веб-клиентом (игроки заходят на https://АЙПИ)
python3 localbox_launcher.py -ip=АЙПИ_СЕРВЕРА -no-web    # только игровой сервер, без раздачи сайта
```

Команда сама: сгенерирует сертификат под этот IP, пропишет `config.json`, выдаст node права на порты 80/443 (через `sudo`/`pkexec`), поставит зависимости и запустит движок. Лог — в терминал, остановка — `Ctrl+C`.

**Если права на порты не выдались автоматически:**

```sh
sudo setcap cap_net_bind_service=+ep "$(command -v node)"
```

**Сертификат вручную** (если нужно):

```sh
mkcert -install
mkcert -cert-file certs/localbox.pem -key-file certs/localbox-key.pem АЙПИ_СЕРВЕРА localhost 127.0.0.1
```

## Подключение игры (хост)

В Steam → свойства игры → **Параметры запуска**:

```
-jbg.config serverUrl=АДРЕС_СЕРВЕРА
```

(`127.0.0.1` для локальной игры или IP сервера в сети). hosts-файл и VPN не нужны.

## Подключение игроков

Открыть в браузере **`https://АДРЕС_СЕРВЕРА`**, ввести код комнаты и имя.

- Локально (один ПК): `https://127.0.0.1`.
- С телефонов/других устройств: `https://IP_сервера` — устройство должно **доверять** сертификату (с mkcert установи корневой CA на устройство; с самоподписанным — прими предупреждение).

## Порты

Движок слушает **80, 443, 38202, 38203**. На Linux порты 80/443 требуют прав (`setcap`/root); на Windows — не требуют.

Если при старте `EADDRINUSE :80` / `:443` — порт занят другой программой:

```sh
# Linux:
sudo ss -ltnp | grep -E ':(80|443)\b'   # найти процесс (nginx/caddy/apache)
sudo systemctl stop nginx caddy apache2  # остановить нужное
```

На Windows — закрой IIS/Skype или запусти лаунчер от администратора (`netstat -ano | findstr :80`).

Если `EADDRINUSE :38203` — висит старый движок:

```sh
sudo pkill -9 -f server/server.js; sudo fuser -k -9 38203/tcp
```

## Английский / русский интерфейс

Лаунчер берёт основным русский `client-ru/` (если есть), иначе английский `client/`. Недостающие в русском файлы автоматически берутся из `client/`, а фирменные ассеты jackbox.ru (задники, логотипы) **докачиваются с jackbox.ru и кэшируются**. Чтобы принудительно использовать английский — убери/переименуй `client-ru/`.

## Как добавить новую игру (например The Jackbox Party Pack 12)

Движок **сам регистрирует любую игру** при создании комнаты (берёт `appId`, `appTag`, число игроков прямо из запроса). Поэтому:

1. **Подключение работает сразу** — ничего в движке менять не надо, даже для новых паков.
2. Нужны только **клиентские ассеты** игры для телефонов. Они подтягиваются автоматически:
   - если jackbox.ru/jackbox.tv уже поддерживают пак — онлайн-докачка возьмёт недостающее при первой загрузке (интернет один раз, потом кэш);
   - либо положи папку игры вручную в `client/` и/или `client-ru/`.

**Явно прописать игру** (задать лимиты игроков) — по желанию, в `server/games.json`:

```jsonc
"appTags":    { "новый-тег": "APP-ID-игры", ... }
"appIds":     { "APP-ID-игры": "новый-тег", ... }
"maxPlayers": { "новый-тег": 8, ... }
"minPlayers": { "новый-тег": 1, ... }
```

`appId` и `appTag` видно в **логе движка** при запуске игры (строка `POST /api/v2/rooms`). После правки `games.json` перезапусти движок.
