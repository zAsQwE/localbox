# Додо Ре Ми (nopus-opus) — офлайн-рендер выступления

Игра **Dodo Re Mi** (The Jackbox Party Pack 10) после записи нот отправляет данные
выступления на серверы Jackbox, где из них собирается **mp3** («выступление птиц»), а хост его
проигрывает. В офлайне этих серверов нет, и без рендера игра **вечно висит** на
«Waiting while performance processes».

LocalBox собирает этот mp3 **локально** через `ffmpeg`: раскладывает сэмплы выбранного
инструмента по временам сыгранных нот, подмешивает бэкинг песни и промахи (flubs), сводит в один
трек и отдаёт хосту. Всё в [server/lib/render.js](../server/lib/render.js).

> ⚠️ Экспериментально. Тембр синтезаторных инструментов приблизительный (генерируется тоном).

---

## Требования

- **ffmpeg** в `PATH` (`ffmpeg -version`). Без него рендер не работает.
  - Arch/CachyOS: `sudo pacman -S ffmpeg` · Debian/Ubuntu: `sudo apt install ffmpeg`
- Node.js (как для всего движка).
- Интернет **один раз** — недостающие сэмплы инструментов докачиваются с
  `cdn.jackboxgames.com` и кэшируются локально (дальше офлайн).

---

## Ассеты (что и куда положить)

Всё лежит в `server/render/` (папка в `.gitignore` — ассеты не наши, большие):

```
server/render/
  nopus-opus/
    instruments/<slug>/<name>.ogg   ← сэмплы инструментов (докачиваются сами; можно положить руками)
    songs/<slug>/backing.ogg         ← БЭКИНГ песни (нужно положить руками!)
    songs/<slug>/backing_<locale>.ogg  (необязательно; локаль-специфичный бэкинг)
  out/                               ← сюда пишутся готовые mp3 (создаётся сам)
```

- **Инструменты** (`instruments/`) движок докачивает сам при первом рендере. Класть вручную не
  обязательно, но можно (ускоряет первый рендер).
- **Бэкинги песен** (`songs/<slug>/backing.ogg`) **нужно положить самому** — их не докачать.
  Возьми из установленной игры: `.../games/NopusOpus/songs/<slug>/` (файл бэкинга) или из своих
  ассетов. `<slug>` — слаг песни (виден в логе рендера и в `payload.songSlug`).

Если бэкинга для песни нет — рендер соберётся **без него** (только инструмент + промахи), просто
тише и без музыки.

## Настройка громкости (переменные окружения)

| Переменная                  | По умолч. | Что делает                                              |
| --------------------------- | --------- | ------------------------------------------------------- |
| `LOCALBOX_RENDER_INSTR`     | `4`       | Усиление инструмента (иначе тонет в бэкинге). Больше — громче. |
| `LOCALBOX_RENDER_BACKING`   | `0.45`    | Громкость бэкинга (0…1). Меньше — тише музыка.          |

Пример запуска с настройкой:

```sh
cd launcher
LOCALBOX_RENDER_INSTR=5 LOCALBOX_RENDER_BACKING=0.4 python3 localbox_launcher.py -ip=АЙПИ_СЕРВЕРА
```

Финальный микс проходит через `alimiter` (защита от клиппинга), так что перегрузить сложно.

## Как это работает (кратко)

1. Хост шлёт `external-request/create` с ключом `render:N`, `service: "song-render"` и `payload`
   (инструмент, ноты, промахи, длительность, слаг песни).
2. Движок синтезирует mp3 (`server/render/out/<id>.mp3`), отдаёт его по `GET /render/<file>`.
3. Движок обновляет сущность `render:N` (тип **`external-request`**) значением:
   ```json
   { "service": "song-render", "status": "success", "progress": 1,
     "response": { "gameRenderUrl": "https://АЙПИ/render/<file>.mp3" } }
   ```
   Формат снят прямо из кода игры (`NopusOpus.swf` → `Playback.handleActionWaitForRendering`):
   хост при `status == "success"` берёт ссылку из **`response.gameRenderUrl`**, качает трек и
   проигрывает, рисуя птиц. (Сначала шлётся `status:"pending"`, затем `success`.)

Скорость: ~3–4 сек на выступление (сэмплы качаются параллельно). У хоста таймаут на рендер —
`renderingDurationAbort` = **120 сек**, так что времени с запасом.

## Диагностика

Запуск с полным логом:

```sh
cd launcher
LOCALBOX_DEBUG=1 python3 -u localbox_launcher.py -ip=АЙПИ 2>&1 | tee ~/localbox-full.log
```

Что искать в логе после игры:

- `[render] запрос: key=render:N service=song-render …` — хост запросил рендер.
- `[render] инструмент <slug> (…)` — какой инструмент (сэмплы / барабаны / синтезатор).
- `[render] готово: https://…/render/…mp3` — mp3 собран.
- **`GET /render/…mp3`** — **хост скачивает трек** (значит рукопожатие прошло, будет звук).
  Если этой строки нет — хост не принял результат.

Частые проблемы:

| Симптом                                   | Причина / решение                                                  |
| ----------------------------------------- | ------------------------------------------------------------------ |
| `[render] нет ffmpeg`                      | Поставь ffmpeg.                                                    |
| Висит на «Waiting while performance processes», нет `[render] готово` | Рендер упал — смотри строку ошибки ffmpeg в логе. |
| Есть `[render] готово`, но нет `GET /render` | Хост не докачивает — проверь, что серверный серт доверен на машине хоста (иначе `<audio>` не грузит). |
| Инструментов почти не слышно              | Подними `LOCALBOX_RENDER_INSTR`, опусти `LOCALBOX_RENDER_BACKING`. |
| Слышны только промахи, нет нот            | Барабаны/синтезатор — должно работать; если нет, пришли лог рендера. |
```
